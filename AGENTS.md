# AGENTS.md

## 项目定位

- 这是一个以原生 `HTML + CSS + JavaScript` 为主的静态前端项目。
- 默认目标是“打开 `index.html` 即可运行”，不引入构建步骤，除非后续任务明确要求。
- 页面应优先面向教学演示、可视化表达和轻量交互场景。

## 开发约束

- 优先使用原生 Web 能力，不默认引入框架、打包器或大型第三方库。
- 新增功能时，先复用现有结构和样式变量，避免重复造轮子。
- 未经明确要求，不接入远程 CDN、外部字体或在线依赖。
- 公式、数学符号和推导展示默认统一使用本地 vendored 的 `KaTeX` 实现，不再混用手写 `sub/sup`、HTML 实体和多套公式风格。
- 保持文件职责清晰：
  - `index.html` 负责页面结构与语义化内容。
  - `styles.css` 负责视觉样式、布局、动画和响应式。
  - `script.js` 或页面专属 `*.js` 文件负责状态、事件和交互逻辑。
  - `math.js` 负责 KaTeX 初始化与共享数学渲染辅助。
  - 静态资源统一放入 `assets/` 目录。
- 避免在 HTML 中写大量内联样式或内联脚本。
- 避免无意义的全局变量；交互逻辑应尽量模块化、可读、可维护。

## 设计与交互要求

- 视觉风格应明确，不做模板化、过于普通的默认页面。
- 默认视觉基调为白色主调的简约风格：
  - 以浅色背景、大面积留白、细线边框和低饱和配色为主。
  - 阴影、渐变和装饰元素应克制，只用于建立层级，不做厚重炫技风格。
  - 若任务需要明显偏离该基调，应由需求明确提出后再调整。
- 使用 CSS 变量统一管理颜色、间距、阴影、圆角和动效参数。
- 页面需同时适配桌面端与移动端。
- 键盘可访问性是默认要求：
  - 可点击元素优先使用原生 `button`、`a`。
  - 支持 `Tab` 焦点导航。
  - 弹窗/面板需支持 `Esc` 关闭。
- 动效要克制且有意义，优先服务于层级表达和状态切换。

## 内容与代码风格

- 默认使用 UTF-8 编码。
- 文案默认使用中文；涉及术语时可保留标准英文名称。
- 类名和变量名使用语义化英文，避免缩写堆叠。
- 注释只写“为什么”，不写显而易见的“做了什么”。
- 删除未使用的代码、样式和资源，避免仓库逐步失控。

## 变更原则

- 修改现有文件前，先理解其职责，不做无关重构。
- 若引入新的交互模式，应同步考虑空状态、异常状态和移动端表现。
- 若新增页面或入口，需要补充简短说明，保证后来者可以快速打开和验证。
- 提交结果至少应满足：
  - 页面可直接打开。
  - 主要交互可用。
  - 结构清晰，便于继续迭代。
